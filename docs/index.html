其他功能 · AI测试后台管理

      
该模块用于支撑 AI测试五大业务功能（文档、案例、管理、执行、结果）的配置、治理与运维。

      

        
五大功能支撑清单

        

          
          

            AI测试功能
            后台管理能力
            目标
            状态
          

          
          
          

            1. 文档
            文档模板管理 / 字段映射管理
            统一文档生成口径，减少手工维护
            已启用
          

          

            2. 案例
            案例生成配置 / 参数规则管理
            提高生成质量，减少不可执行案例
            已启用
          

          

            3. 管理
            案例状态流转策略（待采纳/已入库）
            支撑审核式入库与回退流程
            已启用
          

          

            4. 执行
            执行策略管理（串并行/超时/重试）
            提升执行稳定性与可观测性
            已启用
          

          

            5. 结果
            结果清理策略 / 归档策略
            控制存储成本并保留关键审计记录
            已启用

项目概述

      
当前页面是 GitHub Pages 演示环境，独立展示 5.py 的经典接口自动化测试平台。

      

      

        
一、总体架构


+-------------------+        +-----------------+        +--------------------+       +-------------------------------+
| Frontend (UI)     | <----> | API Controller  | <----> | Test Case Service  | <----> | Database (MySQL/PostgreSQL)  |
+-------------------+        +-----------------+        +--------------------+       +-------------------------------+
                                                          ↑                   ↑
                                                          |                   |
                                              +------------------+     +-----------------+
                                              | LLM Model Client |     | Task Execution  |
                                              +------------------+     +-----------------+

故障处理

      
    
        
  • 接口新增失败：检查 funcNo + sysId 唯一性。
    • 
        
  • 案例新增失败：检查 caseId + sysId 唯一性。
    • 
        
  • hisId 查询失败：确认执行任务是否已创建并完成。
    • 
        
  • 紧急恢复流程：停止流量 -> 切换备用版本 -> 回滚上一个稳定包。
    • 
      

    


    

      
安全措施

      
    
        
  • 数据加密：敏感字段脱敏存储。
    • 
        
  • 访问控制：建议接入统一认证中心。
    • 
        
  • 防火墙设置：限制来源网段。
    • 
        
  • 安全审计：定期导出执行日志与异常日志。
