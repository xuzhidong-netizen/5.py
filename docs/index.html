项目概述

      
当前页面是 GitHub Pages 演示环境，独立展示 5.py 的经典接口自动化测试平台。

      

      

        
一、总体架构


+-------------------+        +-----------------+        +--------------------+       +-------------------------------+
| Frontend (UI)     | <----> | API Controller  | <----> | Test Case Service  | <----> | Database (MySQL/PostgreSQL)  |
+-------------------+        +-----------------+        +--------------------+       +-------------------------------+
                                                          ↑                   ↑
                                                          |                   |
                                              +------------------+     +-----------------+
                                              | LLM Model Client |     | Task Execution  |
                                              +------------------+     +-----------------+

故障处理

      
    
        
  • 接口新增失败：检查 funcNo + sysId 唯一性。
    • 
        
  • 案例新增失败：检查 caseId + sysId 唯一性。
    • 
        
  • hisId 查询失败：确认执行任务是否已创建并完成。
    • 
        
  • 紧急恢复流程：停止流量 -> 切换备用版本 -> 回滚上一个稳定包。
    • 
      

    


    

      
安全措施

      
    
        
  • 数据加密：敏感字段脱敏存储。
    • 
        
  • 访问控制：建议接入统一认证中心。
    • 
        
  • 防火墙设置：限制来源网段。
    • 
        
  • 安全审计：定期导出执行日志与异常日志。
    •
