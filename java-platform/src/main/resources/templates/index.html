项目概述

            
这是一个经典的接口测试自动化平台实现，重点覆盖你手册里的业务流程：案例、接口、版本查询、智能体执行、DevOps执行、517268总线分发、状态和结果查询。

            

            

                
一、总体架构

                
架构图

                
+-------------------+        +-----------------+        +--------------------+       +-------------------------------+
| Frontend (UI)     | <----> | API Controller  | <----> | Test Case Service  | <----> | Database (MySQL/PostgreSQL)  |
+-------------------+        +-----------------+        +--------------------+       +-------------------------------+
                                                          ↑                   ↑
                                                          |                   |
                                              +------------------+     +-----------------+
                                              | LLM Model Client |     | Task Execution  |
                                              +------------------+     +-----------------+

                
组件分析

                
    
                    
  • Frontend (UI)：输入接口文档/参数并触发生成与执行。
    • 
                    
  • API Controller：接收请求并转发到核心服务。
    • 
                    
  • Test Case Service：构造Prompt、调用LLM、解析并入库。
    • 
                    
  • Database：保存测试用例、任务与执行结果。
    • 
                    
  • LLM Model Client：对接大模型接口生成测试用例。
    • 
                    
  • Task Execution：执行生成用例并返回执行结果。
    •

故障处理

            
    
                
  • 接口新增失败：检查 funcNo + sysId 唯一性。
    • 
                
  • 案例新增失败：检查 caseId + sysId 唯一性。
    • 
                
  • hisId 查询失败：确认执行任务是否已创建并完成。
    • 
                
  • 紧急恢复流程：停止流量 -> 切换备用版本 -> 回滚上一个稳定包。
    •
